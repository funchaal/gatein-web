# Dark Mode Switch Implementation

Add a dark mode toggle to the sidebar and adapt all screens/components to correctly render in both light and dark modes.

## Background

The project already has solid foundations for dark mode:
- [index.css](file:///c:/Users/rafae/Documents/GateIn/web/gatein-webapp/src/index.css) already defines CSS custom properties for both `:root` (light) and `.dark` (dark) selectors (lines 45–114), with `@custom-variant dark (&:is(.dark *))` on line 4
- [globalSlice.js](file:///c:/Users/rafae/Documents/GateIn/web/gatein-webapp/src/store/slices/globalSlice.js) already has `toggleTheme`/`setTheme` Redux actions that toggle the `.dark` class on `<html>` and persist to `localStorage`
- The Redux store already registers the `global` reducer in [store/index.js](file:///c:/Users/rafae/Documents/GateIn/web/gatein-webapp/src/store/index.js)

**The problem**: None of this is wired up in the UI, and virtually every component uses hardcoded Tailwind `bg-white`, `text-gray-900`, `bg-gray-50`, `border-gray-200`, etc. instead of the CSS variable-backed semantic classes (`bg-background`, `text-foreground`, `bg-card`, `text-card-foreground`, `bg-muted`, `border-border`, etc.).

> [!IMPORTANT]
> **Strategy for inline styles vs Tailwind**: Many screens (Home, NotFound, LoginPage) use extensive **inline styles** with hardcoded hex colors. For these, I'll use a CSS class approach or conditional dark-mode styles via the `useTheme` hook, since Tailwind `dark:` variants don't work with inline `style={}` attributes.

> [!NOTE]
> **Preview panel exception**: The phone mockup previews in the LayoutEditor (CardPreview, ModalPreview, LayoutPreviewPanel) simulate mobile app rendering and should NOT be affected by dark mode — they always show the light app UI as it would appear on the user's phone.
